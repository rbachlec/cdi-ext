    cdi-ext - Some extensions for CDI

    Here I would like to describe the features of my tiny cdi-ext library and how you could use them in your project.

    The main features are:

    • FactoryExtension: To be used if producer methods are not sufficient
    • DisposableSingleton: Works like ApplicationScoped beans but beans can be disposed manually
    • AnnotationUtil: Query annotations on AnnotatedType instances
    • BeanManagerUtil: Get contextual instances and inject things in unmanaged beans

    FactoryExtension

    The reason for this CDI extension is the lack of generic producer methods in CDI. You can define qualifiers to call different producer methods on instantiation but you cannot write a producer method responsible for the instantiation of different classes (even if they should be created in the same way).

    Usage

    1. Create a factory by implementing the Factory interface.
    2. Annotate classes to be created by the factory with @ProducedByFactory.

    DisposableSingleton scope

    You need singletons in your CDI application (like ApplicationScoped beans) but you want to drop them at times? Then this extension is the right thing for you.

    Usage

    1. Annotate your singletons with @DisposableSingleton.
    2. You can drop the singleton from the context by injecting the DisposableSingletonContext and call the disposeSingleton() method.
    3. But be careful: Already injected instances are not replaced automatically with a new instance on usage.
